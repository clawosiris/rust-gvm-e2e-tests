# 2026-03-23 — E2E Test Repo Creation & Phase 1 Implementation

## Background

The E2E test infrastructure originally lived inside `clawosiris/rust-gvm` under `tests/e2e/gvm-community/`. As the ecosystem grew (rust-gvm, gvm-rools, rust-gvm-api, openvas-mcp-server), we decided to extract it into a standalone repo so it could test all components against a single GVM stack.

## Decision: Standalone E2E Repo

**Discussion participants:** Daniel Riek, y0urself (Jaspar), Thoth

**Key arguments for standalone:**
- Tests all interface layers in one place (library, CLI, REST, gRPC, MCP)
- No circular dependency awkwardness (rust-gvm E2E tests used gvm-rools as a dev-dep)
- Independent evolution — compose changes don't pollute component repos
- Single runner allocation for all tests
- Cross-repo triggering via `repository_dispatch`

**Architecture decided:**
- Phase 1: Library (rust-gvm) + CLI (gvm-rools) — Unix socket only
- Phase 2: REST/gRPC API + SSH transport (when rust-gvm-api is ready)
- Phase 3: MCP server (when openvas-mcp-server is production-ready)
- Phase 4: Cross-client differential validation (optional)

## History-Preserving Migration

Used `git-filter-repo` to extract E2E-relevant paths from a mirror clone of rust-gvm, preserving commit history:

```bash
git clone --mirror clawosiris/rust-gvm rust-gvm.mirror
git clone rust-gvm.mirror rust-gvm-split
cd rust-gvm-split
git checkout fix/e2e-runner-labels-v2
git filter-repo \
  --path tests/e2e/gvm-community/ \
  --path .github/workflows/e2e.yml \
  --path examples/e2e_gvm_community.rs \
  --path-rename tests/e2e/gvm-community/:docker/ \
  --path-rename examples/e2e_gvm_community.rs:rust/examples/e2e_gvm_community.rs
git push --force e2e HEAD:main
```

Result: 9 commits preserved from the original E2E harness development.

## Migration Issues Encountered

Moving from "in-repo" to "standalone" caused several failures, all one-time migration costs:

| Issue | Cause | Fix |
|-------|-------|-----|
| Docker build can't fetch rust-gvm | Private repo, no auth in Dockerfile | BuildKit `--mount=type=secret` with RELEASE_TOKEN |
| Runner label mismatch | New runner missing `docker` label | Changed workflow to `[self-hosted]` |
| wait-ready.sh checking host filesystem | Socket only exists inside container | Rewrote to use `docker compose exec` |
| `cargo run` in smoke script | No cargo in runner image (pre-built binary) | Call `gvm-community-e2e` directly |
| PATH not set | `/usr/local/cargo/bin` not in entrypoint PATH | Explicit `ENV PATH` in Dockerfile |
| Missing port_list_id | GMP requires PORT_LIST for create_target | Pick first port list from get_port_lists |
| CLI `--socket-path` vs `--path` | gvm-rools renamed the flag | Updated smoke.sh |
| CLI flag ordering | `--xml` must come before `socket` subcommand | Refactored `run_cli()` helper |
| Codex branch force-push | Squash merge + Codex push overwrote fixes | Force-push with correct history |
